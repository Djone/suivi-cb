Parcours recommandé

  1. 1. dry-run pour vérifier (Git + tests) sans modifier la release.
  2. 2. prepare pour clôturer la version actuelle et préparer la suivante.
  3. 3. deploy pour la mise en production (avec execute coché).

Option rapide: full = prepare + deploy en une seule exécution.

Notes (définitions simples)

Modes d'exécution

  • dry-run: vérifie sans modifier.
  • prepare: met à jour les versions.
  • deploy: déploie (si execute est coché).
  • full: prepare + deploy.

Options de contrôle

  • allow-dirty: autorise les fichiers non commit.
  • skip-master-check: ignore le contrôle de synchro master.
  • skip-tests: ignore les tests.
  • skip-build: ignore le build pendant deploy.
  • execute: lance les actions réelles (sinon simulation).

Git et rollback

  • create-release-branch: crée une branche release.
  • commit: commit automatique des fichiers de version.
  • tag: crée le tag vX.Y.Z.
  • rollback-on-failure: restaure automatiquement en cas d'erreur.
  • Rollback manuel: restaure depuis le dernier backup.
